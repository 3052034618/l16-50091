## 1. 产品概述

开发流水线状态可视化看板——一个面向研发团队的实时 DevOps 仪表盘，通过 GitHub Webhook 接收仓库事件（Push、Pull Request、GitHub Actions），在看板上以仓库分组展示 PR 列表、CI 检查状态、代码审查意见和合并状态；流水线视图按阶段（构建→测试→部署）展示各步骤执行状态与耗时；支持测试覆盖率趋势图、一键触发 GitHub Actions 工作流，以及部署频率与成功率统计。所有数据实时推送更新，无需刷新页面。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 开发者 | GitHub OAuth 登录 | 查看看板、触发工作流 |
| 管理员 | GitHub OAuth 登录 + 权限提升 | 配置 Webhook、管理仓库 |

### 2.2 功能模块

1. **看板主页**：按仓库分组展示最近 PR 列表，内联显示 CI 检查状态、审查人意见和合并状态
2. **流水线视图**：按阶段（构建→测试→部署）展示各步骤执行状态和耗时
3. **测试覆盖率页**：测试覆盖率历史趋势图，按分支汇总展示
4. **工作流触发页**：一键触发 GitHub Actions 中指定的工作流（如重新部署）
5. **部署统计页**：按仓库和时间范围的部署频率和成功率统计

## 3. 核心流程

1. **Webhook 注册流程**：管理员选择仓库 → 系统调用 GitHub API 创建 Webhook → Webhook 回调地址指向本系统 → 开始接收事件
2. **PR 状态更新流程**：GitHub 推送 Webhook 事件 → 后端解析事件 → 更新数据存储 → WebSocket 推送到前端 → 看板实时刷新
3. **工作流触发流程**：用户点击触发 → 调用 GitHub API 创建 Workflow Dispatch → 轮询/Webhook 获取执行状态 → 实时同步到前端
4. **统计查询流程**：用户选择仓库和时间范围 → 后端从数据存储聚合 → 返回统计数据 → 前端图表渲染

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深色主题（#0F1117 背景），搭配翠绿色（#10B981）作为成功/运行态指示，琥珀色（#F59E0B）作为等待态，红色（#EF4444）作为失败态
- **辅助色**：蓝紫色（#8B5CF6）用于交互高亮和品牌元素
- **按钮风格**：圆角按钮，主要操作用实心色，次要操作用边框色
- **字体**：JetBrains Mono 用于代码和状态文本，DM Sans 用于标题和正文
- **布局风格**：左侧导航栏 + 主内容区，卡片式布局，顶部状态栏
- **图标**：Lucide 图标库
- **动效**：卡片进入时淡入滑入，状态变更时脉冲高亮，加载时骨架屏

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 看板主页 | 仓库筛选栏 | 顶部横条，仓库图标+名称 Chip，搜索输入框，WebSocket 连接状态点 |
| 看板主页 | PR 列表卡片 | 仓库分组标题，卡片网格布局，每卡含 PR 标题、作者头像、CI 徽章、审查人头像行、合并状态标签 |

### 4.3 响应式设计

- 桌面优先设计，1280px 及以上完整展示
- 平板（768-1279px）侧边栏收缩为图标模式，卡片从三列变两列
- 移动端（<768px）底部标签导航，单列卡片布局
